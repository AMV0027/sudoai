# Contributing to sudoai

First off, thank you for considering contributing to `sudoai`! It's people like you that make the open-source community such a great place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker to report bugs.
- Include a clear, descriptive title and a detailed description of the issue.
- Provide steps to reproduce the bug.
- Mention your OS and Node.js version.

### Suggesting Enhancements
- Open an issue with the "enhancement" label.
- Explain why this feature would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Issue that pull request!

## Style Guide

- We use TypeScript for all core logic.
- Follow the existing project structure in `src/`.
- Use `zod` for any new tool or configuration schemas.
- Document new features in the `README.md`.
